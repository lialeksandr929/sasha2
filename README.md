# GitHub User Finder

**Автор:** Саша Ли

## Описание

GitHub User Finder — это десктопное приложение на Python с графическим интерфейсом (GUI), позволяющее:
* искать пользователей GitHub по имени;
* просматривать основную информацию о пользователе (логин, имя, местоположение, количество публичных репозиториев);
* добавлять пользователей в список избранного;
* просматривать список избранных пользователей.

Приложение использует GitHub API для получения данных о пользователях и сохраняет избранное в локальный файл `favorites.json`.

## Как использовать

### Требования

* Python 3.6 или выше;
* установленные библиотеки: `tkinter` (обычно входит в стандартную поставку Python), `urllib`, `json`, `os` (также стандартные библиотеки).

### Запуск приложения

1. Сохраните код приложения в файл с именем `github_user_finder.py`.
2. Убедитесь, что файл находится в рабочей директории (рядом с ним будет создан файл `favorites.json`).
3. Запустите приложение из командной строки:
   ```bash
   python github_user_finder.py
Интерфейс и функционал
Поле поиска: введите имя пользователя GitHub и нажмите кнопку «Найти» или клавишу Enter.

Результаты поиска: информация о найденном пользователе отобразится в таблице ниже.

Добавить в избранное: выберите строку с пользователем в таблице и нажмите кнопку «Добавить в избранное». Пользователь сохранится в локальном файле.

Показать избранное: нажмите кнопку «Показать избранное», чтобы отобразить список всех ранее добавленных пользователей.

Примеры использования
Сценарий 1: Поиск существующего пользователя
Введите в поле поиска, например, octocat (это тестовый аккаунт GitHub).

Нажмите «Найти».

В таблице должны появиться данные: логин octocat, имя The Octocat, местоположение и количество публичных репозиториев.

Сценарий 2: Попытка найти несуществующего пользователя
Введите заведомо несуществующий логин, например, thisuserdoesnotexist12345.

Нажмите «Найти».

Вы увидите всплывающее окно с ошибкой: «Пользователь ‘thisuserdoesnotexist12345’ не найден!».

Сценарий 3: Работа с избранным
Найдите какого‑либо существующего пользователя (например, octocat) и отобразите его данные.

Выберите строку с этим пользователем в таблице.

Нажмите кнопку «Добавить в избранное». Появится сообщение об успехе.

Нажмите кнопку «Показать избранное». В таблице должен появиться добавленный пользователь.

Попробуйте добавить того же пользователя ещё раз. Вы увидите сообщение: «[логин] уже в избранном!».

Сценарий 4: Проверка работы после перезапуска
Добавьте одного или нескольких пользователей в избранное.

Закройте приложение.

Снова запустите github_user_finder.py.

Нажмите «Показать избранное». Список избранных пользователей должен загрузиться из файла favorites.json.

Возможные ошибки и их устранение
Ошибка сети: если нет подключения к интернету или GitHub API недоступен, появится сообщение «Не удалось подключиться к GitHub API». Проверьте подключение к сети.

HTTP‑ошибки: любые ошибки, кроме 404 (не найден), будут показаны в формате «HTTP [код]: [причина]».

Пустое поле поиска: приложение не позволит выполнить поиск, если поле ввода пустое, и выдаст соответствующее предупреждение.

Примечание: файл favorites.json создаётся автоматически при первом добавлении пользователя в избранное. Его можно удалить, чтобы сбросить список избранного.
